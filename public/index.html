OpenShift PaaS Demo

Team: Rina Ismaili und Alma Dacik. Diese Seite zeigt live: Die App läuft wirklich in OpenShift (Pods + Route) und liefert echte Daten aus dem Cluster.

Start Tests Kurz erklärt

Was bedeutet das hier ganz einfach?

  • OpenShift ist wie ein „App-Manager“ für Container.
  • Du sagst: „Meine App soll laufen“ – OpenShift startet sie automatisch.
  • Die App läuft in sogenannten Pods (das sind laufende App-Instanzen).
  • Über eine Route bekommst du einen Link, den du im Browser öffnen kannst.

Wichtig für unseren Prof: Die Werte rechts kommen live aus dem laufenden Pod. Damit sieht man eindeutig: Das ist nicht nur eine statische Webseite, sondern eine App im Cluster.

Wenn du 2 Pods einstellst und öfter neu lädst, kann der Pod-Name wechseln. Das ist normal und zeigt, dass OpenShift die Anfragen verteilt.

Merksatz: Pod-Name + Namespace + Uptime = „Das ist ein laufender Container in OpenShift“.

So zeigst du es dem Prof (sehr klar)

  • 1) Seite öffnen: Du siehst Pod + Namespace + Uptime.
  • 2) In OpenShift Replicas von 1 auf 2 stellen.
  • 3) Hier 30 Requests starten: Wenn 2 Pods erscheinen, ist Load-Balancing sichtbar.
  • 4) APP_VERSION von v1 auf v2 ändern (oder Git push) → neue Version wird ausgerollt.
  • 5) Optional: Einen Pod löschen → OpenShift startet automatisch einen neuen (Self-Healing).
